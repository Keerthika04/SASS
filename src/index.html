What is Sass?

Sass (Syntactically Awesome Style Sheets) is a preprocessor scripting language that is interpreted into Cascading Style Sheets (CSS). It adds special features to CSS, such as variables, nested rules, mixins, maps, and inheritance, making it easier to manage styles in large projects.


Key Features of Sass

  • Variables: Store values like colors, fonts, or any CSS value to reuse throughout your stylesheet.
  • Nested Rules: Write CSS in a nested manner, making it more readable and organized.
  • Mixins: Create reusable chunks of CSS that can include variables and other mixins.
  • Partials: Split your CSS into smaller, manageable files and include them in a main stylesheet.
  • Inheritance: Use the @extend directive to share a set of CSS properties from one selector to another.
  • Maps: Use key-value pairs to store related values, like theme colors, making it easier to manage and update.

Getting Started with Sass

To use Sass, you need to install it. You can do this via npm:

npm install sass --save-dev

Once installed, you can compile your Sass files to CSS using a command like the following:

sass input.scss output.css

If you want to automate your Sass compilation process, you can use a command in your package.json file. Hereâ€™s an example command: 

"start": "sass ./Sass/style.scss ./Css/style.css && sass ./Sass/style.scss ./Css/style.min.css --style=compressed --no-source-map --watch"

This command does the following:

  • Compiles: It compiles style.scss from the Sass directory to style.css in the Css directory.
  • Minifies: It also creates a minified version, style.min.css, which is optimized for production.
  • Watches for Changes: The --watch flag allows it to automatically recompile when you make changes to your Sass files.
  • Compresses: The --style=compressed option reduces the file size of the output CSS.
  • No Source Maps: The --no-source-map flag prevents source map files from being generated, keeping the output cleaner.

Variables

Variables in Sass allow you to store values like colors, fonts, or any CSS property for reuse throughout your stylesheets. This reduces repetition and makes it easier to manage changes.


// Define variables
$primary-color: #3498db;
$secondary-color: #2ecc71;
$font-stack: Helvetica, sans-serif;

// Use variables
body {
  font-family: $font-stack;
  color: $primary-color;
}

button {
  background-color: $secondary-color;
  color: white;
}

Maps

Maps in Sass allow you to store key-value pairs, making it easier to group related values like theme colors or breakpoints.


// Define a map of theme colors
$theme-colors: (
  primary: #3498db,
  secondary: #2ecc71,
  warning: #e74c3c,
  info: #8e44ad
);

// Accessing values from the map
button {
  background-color: map-get($theme-colors, primary);
  color: white;
}

.alert-warning {
  background-color: map-get($theme-colors, warning);
  color: white;
}

This example shows how to create a map called $theme-colors and use the map-get function to access the colors inside it.

Nested Rules

Nesting allows you to structure your CSS in a way that mimics the HTML structure. This improves readability and keeps related styles together.


nav {
  ul {
    list-style: none;
  }

  li {
    display: inline-block;
    margin-right: 10px;
  }

  a {
    text-decoration: none;
    color: $primary-color;

    &:hover {
      color: $secondary-color;
    }
  }
}

Inheritance (@extend)

With the @extend directive, you can share a set of styles between selectors, avoiding repetition in your code.


// Base button style
.button {
  padding: 10px 20px;
  border: none;
  cursor: pointer;
}

// Extend the button style
.primary-button {
  @extend .button;
  background-color: $primary-color;
  color: white;
}

.secondary-button {
  @extend .button;
  background-color: $secondary-color;
  color: white;
}

Mixins

Mixins allow you to create reusable chunks of code that can be included in other rules. You can even pass arguments to them to customize the output.


// Use of Mixin
@mixin rounded-corners($radius) {
    border-radius: $radius;
    border: none;
}

// Use of extend and Mixin
.btn {
    @include rounded-corners(5px);
    padding: 10px 20px;
    border: none;
    cursor: pointer;
}

// Use of Mixin
.mixin {
    @include rounded-corners(5px);
    padding: 10px 20px;
    background-color: map-get($colors, info);
    color: map-get($colors , dark);
}

.extend {
// Use of extend
    @extend .btn;
    background-color: map-get($colors, success);
    color: map-get($colors , light);
}

Partials

Partials let you split your Sass into smaller, more manageable files, which are then imported into a main stylesheet. This helps in organizing your code.


// _variables.scss
$primary-color: #3498db;
$font-stack: Helvetica, sans-serif;

// _buttons.scss
button {
  background-color: $primary-color;
  font-family: $font-stack;
}

// main.scss
@import 'variables';
@import 'buttons';
